CHKDSK для ZX Sprinter / Estex DSS
==================================

(c) 2026 Dmitry Mikhalchenkov, Sprinter Team. Лицензия GPLv3.

Утилита проверки и восстановления файловых систем FAT для дискет,
IDE / CF / SD разделов и других носителей, доступных через DSS BIOS.


КАКОЙ БИНАРНИК ЗАПУСКАТЬ
-------------------------

Единый бинарник CHKDSK.EXE покрывает FAT12, FAT16 и FAT32 -- и для
дискет, и для IDE / CF / SD разделов. Тип FAT определяется во время
работы по BPB, выбирать нечего. (В ранних выпусках FAT12 шёл отдельным
CHKDSK12.EXE; это разделение упразднено.)


КОМАНДНАЯ СТРОКА
-----------------

  CHKDSK <буква>: [/F] [/C] [/V] [/Y]

Буква диска обязательна и должна сопровождаться двоеточием, например
C:, D:, A:. Регистр не важен.

  /F     Применять исправления. Без /F программа работает в режиме
         «только чтение» и лишь сообщает о найденных проблемах.

  /C     Совместно с /F: при обнаружении lost cluster'ов каждая
         висячая цепочка превращается в запись FILE####.CHK в корне
         тома вместо тихого освобождения. Полезно для восстановления
         данных. Без /F флаг игнорируется.

  /V     Verbose: точечный индикатор прогресса в долгих фазах
         (Phase 2 -- FAT scan, Phase 3 -- обход дерева каталогов,
         Phase 4 -- sweep). Косметика, не влияет на результат.

  /Y     Совместно с /F: подразумевать «да» на однократном
         предупреждении о записи (см. ВАЖНО ДЛЯ БЕЗОПАСНОСТИ) и
         начинать исправления без ожидания клавиши. Для пакетного /
         автоматического запуска. Без /F не действует.

  /?     Справка.

Долгий проход можно прервать в любой момент клавишей ESC или Ctrl+C:
программа остановится на ближайшей контрольной точке, уже записанные
исправления останутся в силе.


ЧТО ДЕЛАЕТ КАЖДАЯ ФАЗА
-----------------------

  Phase 1 -- boot sector and BPB
    Проверяет boot signature (0xAA55), геометрию (размер сектора и
    кластера), число FAT, количество записей корневого каталога.
    По числу кластеров классифицирует том как FAT12, FAT16 или
    FAT32.

  Phase 2 -- FAT tables
    Обход всех записей FAT. Каждая запись классифицируется: free
    (свободный), in-use (используется), BAD (битый), end-of-chain
    (конец цепочки), invalid (мусор). Если у тома две копии FAT,
    также детектируется их рассинхронизация. FAT[0] (media-байт)
    сверяется с BPB; FAT[1] проверяется на валидный EOC, а на
    FAT16/32 -- на флаги clean-shutdown и hard-error.

  Phase 3 -- directory and chain walk
    Рекурсивный обход всех каталогов от корня. Каждая запись
    проверяется:

      * SFN -- допустимые символы, регистр, ведущий пробел, биты
        атрибутов, согласованность DIR/файл, диапазон first cluster,
        high-cluster word на FAT12/16, требование size==0 для DIR.
      * Указатели "." / ".." в подкаталогах (должны указывать на
        собственный кластер / кластер родителя).
      * Группы LFN текущего формата: порядок и пределы номеров слотов,
        NUL / 0xFFFF padding, единая контрольная сумма внутри группы,
        совпадение этой суммы с коротким именем за ней, запрещённые
        UCS-2 символы, обязательные нулевые поля и «осиротевшие» группы
        без короткого имени. Зарезервированные будущие типы LFN-dirent
        сохраняются и игнорируются. Сами имена не декодируются -- в
        путях всегда печатается имя 8.3.
      * Цепочка кластеров: циклы, cross-link'и, обрывы / BAD,
        truncated (короче чем заявленный size), excess (длиннее
        чем нужно).

    Чистые записи не выводятся. В конце выводится строка Totals с
    подсчётом популяции.

  Phase 4 -- lost clusters
    Все записи FAT, помеченные как in-use, но не достижимые ни из
    одной директории, считаются lost. Только сообщает; чинит
    с /F или /F /C.

После Phase 4 печатается отчёт в стиле классического chkdsk: всего
места на диске, байт в пользовательских файлах и в каталогах, байт в
bad-секторах, байт свободно, и геометрия в единицах распределения.
Выводится только серийный номер тома, без метки -- DSS не отдаёт метку
через структуру монтирования.


ИСПРАВЛЕНИЯ
------------

С флагом /F программа исправляет следующие классы проблем. Каждый
успешный ремонт увеличивает счётчик applied в итоговой сводке;
счётчик found растёт независимо от /F при каждом обнаружении.

  Рассинхрон FAT 1/2
    FAT 1 копируется поверх FAT 2 сектор за сектором.

  Запись каталога ATTR_DIR с ненулевым size
    Если первый кластер записи похож на реальный subdir (первая
    32-байтовая ячейка начинается с записи "."), только size DWORD
    обнуляется. Иначе запись помечается удалённой (0xE5) -- почти
    всегда это файл с искажённым атрибутом, в который ошибочно попал
    бит ATTR_DIR.

  Битые указатели "." / ".."
    В кластерных полях dot-записи прописывается правильное значение
    (собственный first cluster для ".", first cluster родителя для
    "..", либо 0 если родитель -- корень).

  Truncated / broken / cross-link цепочки
    Поле size в записи каталога уменьшается до фактического покрытия
    цепочки, чтобы видимое пользователю содержимое заканчивалось на
    надёжных данных.

  Цепочки excess (длиннее, чем требует size)
    На ожидаемом последнем кластере записывается EOC, хвост цепочки
    освобождается.

  Lost clusters
    По умолчанию: записи FAT обнуляются (становятся free). С /C
    каждая орфан-цепочка связывается в запись FILE####.CHK в корне
    тома, чтобы её содержимое можно было восстановить.

  Недопустимые символы SFN, строчные буквы, ведущий пробел
    Имя 8.3 очищается на месте: запрещённые байты заменяются на '_',
    строчные ASCII и строчная кириллица CP866 переводятся в верхний
    регистр (второе важно потому, что сама DSS не может открыть файл,
    в имени которого хранится «сырая» строчная кириллица). Если перед
    записью стоит корректная группа длинного имени, её контрольная
    сумма переклеймляется под новое короткое имя -- длинное имя
    переживает переименование. Правка пропускается -- с выводом
    причины -- если очищенное имя совпало бы с другой записью в том же
    каталоге или обновлению помешала ошибка I/O. Поддерживаются группы,
    пересекающие границы секторов и кластеров каталога.

  Повреждённые группы длинных имён (LFN)
    Группа с испорченным порядком слотов, padding, контрольной суммой
    или символами удаляется целиком, в том числе через границы секторов
    и кластеров каталога. Длинное имя теряется, но короткое имя 8.3 за
    ней остаётся нетронутым, и файл полностью доступен.
    «Осиротевшая» группа без короткого имени также удаляется.


ЧТО НЕ ИСПРАВЛЯЕТСЯ
--------------------

  * Сбойные участки на уровне носителя. Программа не делает surface
    scan и не помечает нечитаемые кластеры как BAD. Если есть
    подозрение на физический износ -- пользуйтесь диагностикой
    железа.

  * Cross-link сообщается и size второго файла шринкуется, но
    цепочки не расщепляются на разные копии. Кластер остаётся за тем
    файлом, который walker встретил первым.

  * Длинные имена никогда не декодируются и не восстанавливаются.
    Испорченное длинное имя удаляется, а не реконструируется --
    пересоздайте его с хост-системы, если оно нужно. Короткое имя 8.3
    при этом всегда остаётся рабочим.

КОДЫ ВОЗВРАТА
--------------

  0   -- том чист.
  1   -- проблемы найдены, не исправлены (прогон без /F, только чтение).
  2   -- проблемы найдены, все исправлены (прогон с /F).
  3   -- проблемы найдены, часть осталась неисправленной (прогон с /F) --
         причину см. в строках WARN (коллизия имён или ошибка I/O).
  255 -- фатально: том вообще не удалось проверить (неверный диск,
         нечитаемый boot/BPB, нет страничной памяти, ошибка ввода-вывода
         посреди прохода).

В шелле DSS `IF ERRORLEVEL n` истинно, когда код >= n (как в MS-DOS),
поэтому в bat-скрипте проверять коды нужно от старшего к младшему:

  CHKDSK C: /F /Y
  IF ERRORLEVEL 255 GOTO Fatal
  IF ERRORLEVEL 3   GOTO Partial
  IF ERRORLEVEL 2   GOTO AllFixed
  IF ERRORLEVEL 1   GOTO FoundUnfixed
  GOTO Clean


ВАЖНО ДЛЯ БЕЗОПАСНОСТИ
-----------------------

  * Сначала всегда запускайте без /F и убедитесь, что отчёт
    осмысленный. Только после этого -- с /F.

  * Перед первой записью прогон с /F печатает
      WARNING: about to write to disk. Press Y to continue, any other
      key to abort.
    Любая клавиша, кроме Y, отменяет остаток прогона -- больше ничего
    не пишется, а код возврата читается как у прогона «только чтение».
    Флаг /Y пропускает это предупреждение для автоматического запуска.

  * Исправления необратимы. Откатить нельзя. Если данные ценные --
    сделайте бэкап тома перед запуском с /F.

  * На съёмных носителях не отключайте/не сбрасывайте устройство во
    время работы с /F: прерывание Phase 2 FAT 1/2 sync на полпути
    может оставить FAT в худшем состоянии, чем было.

  * Если Phase 4 нашёл много lost clusters -- предпочтите /F /C
    обычному /F: записи FILE####.CHK позволят разобраться, что
    осиротело, прежде чем освобождать пространство.
